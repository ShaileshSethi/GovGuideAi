# 04_UI_UX.md

# UI / UX Guidelines

The interface should be modern, clean, and easy for users of all ages.

The design should prioritize readability and simplicity over unnecessary animations.

---

# Theme

Style:

- Clean
- Minimal
- Professional
- Government-friendly
- AI-powered

Primary Color:
Blue (#2563EB)

Secondary Color:
White

Accent Color:
Green for success
Red for warnings

Use rounded cards, soft shadows, and plenty of spacing.

---

# Home Page

Display:

- Logo
- Project Name
- Tagline
- Large AI search box
- Search button
- Example prompts below the search box

Example Prompts:

- I lost my Aadhaar card
- I need a passport
- I got married
- My child was born
- I want a driving licence

---

# Search Results Page

Display:

1. AI Summary at the top

2. Government Service Cards

3. Document Checklist

4. Step-by-Step Guide

5. Official Links

---

# Service Card

Each card should include:

- Service Name
- Short Description
- Required Documents
- Fees
- Processing Time
- "View Details" button
- "Visit Official Website" button

Cards should be expandable.

---

# Document Checklist

Display required documents as checkboxes.

Example:

☐ Aadhaar Card

☐ PAN Card

☐ Birth Certificate

☐ Passport Photo

Users can manually mark completed items.

---

# Buttons

Primary Button:

- Blue background
- White text

Secondary Button:

- White background
- Blue border

Buttons should have hover effects.

---

# Typography

Use a modern font such as Inter.

Headings should be bold.

Body text should be easy to read.

Avoid long paragraphs.

---

# Responsive Design

Support:

- Mobile
- Tablet
- Desktop

Cards should stack vertically on smaller screens.

Navigation should remain simple and touch-friendly.

---

# Loading State

While the AI is generating a response, show:

- Loading spinner
- Message: "Analyzing your request..."

---

# Empty State

Before any search, encourage users with:

"Describe your situation, and GovGuide AI will create your government action plan."

---

# Error State

If the AI cannot understand the request:

- Display a friendly error message.
- Ask the user to rephrase or provide more details.

---

# Overall Goal

The application should feel as simple as chatting with an AI assistant while presenting government information in a clear, organized, and trustworthy way.
