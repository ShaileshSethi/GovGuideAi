# 07_FRONTEND_STRUCTURE.md

# Frontend Structure

The application should have a clean and simple layout with minimal navigation.

---

# Pages

## 1. Home Page

Purpose:
Allow users to describe their problem.

Components:

- Navbar
- Hero Section
- Search Box
- Search Button
- Example Prompts
- Footer

---

## 2. Results Page

Purpose:
Display the AI-generated government action plan.

Components:

- User Query
- AI Summary
- Government Service Cards
- Document Checklist
- Step-by-Step Guide
- Official Links
- Suggested Searches

---

# Components

## Navbar

Contains:

- Logo
- Project Name

---

## Hero Section

Display:

**GovGuide AI**

"Describe your problem. We'll tell you exactly what to do."

---

## Search Box

A large input field where users can type naturally.

Examples:

- I lost my wallet
- I need a passport
- I got married

Include a prominent **"Generate Action Plan"** button.

---

## AI Summary Card

A short overview generated by the AI.

Example:

"You need four documents to apply for a passport. The process usually takes 2–3 weeks."

---

## Service Card

Each government service should display:

- Service Name
- Description
- Required Documents
- Fees
- Processing Time
- Official Website Button
- Expand for More Details

---

## Document Checklist

Each required document appears as a checkbox.

Example:

☐ Aadhaar Card

☐ PAN Card

☐ Birth Certificate

☐ Passport Photo

---

## Official Link Button

Every service should include a clear button linking to the official government website.

Example:

**Visit Official Website**

---

## Suggested Searches

At the bottom of the page, display related searches.

Examples:

- Passport Renewal
- PAN Card
- Driving Licence
- Aadhaar Update

---

# Responsive Design

The interface must work well on:

- Mobile
- Tablet
- Desktop

Cards should stack vertically on smaller screens.

---

# Design Goal

The user should be able to:

1. Describe their problem.
2. View their personalized action plan.
3. Open the official government website.

The experience should require as few clicks as possible.
