# Supported Government Services

The MVP should support the following common government services.

Each service should have its own knowledge file containing:

- Overview
- Required Documents
- Application Steps
- Fees
- Processing Time
- Official Government Links

## Future Services

The knowledge base should be easy to expand.

To add a new service:

1. Create a new knowledge file.
2. Follow the same structure.
3. Register the service in the application.
4. No other code changes should be required.

---

## Notes

For the hackathon MVP, focus on implementing 10–15 high-demand services with complete and accurate information.

The remaining services can display placeholder data or be added later.
