# 02_FEATURES.md

# Core Features

GovGuide AI converts a user's real-life problem into a complete government action plan.

The application should focus on simplicity and accuracy.

---

## Feature 1 — AI Problem Solver

The home page contains a single large input box.

Users should be able to type naturally.

Examples:

- I lost my Aadhaar card
- I got married
- I want a passport
- My baby was born
- I changed my address
- I want to vote
- I want to study abroad

The AI should understand the user's intent without requiring exact keywords.

---

## Feature 2 — Government Roadmap

Instead of returning only text, generate a roadmap.

Example:

Passport

↓

Required Documents

↓

Apply Online

↓

Track Status

Each step should be shown as a card.

---

## Feature 3 — Service Cards

Every government service should have its own card.

Each card contains:

- Service Name
- Description
- Required Documents
- Processing Time
- Fees
- Official Website
- Apply Button

Cards should be expandable.

---

## Feature 4 — Document Checklist

Every service displays a checklist.

Example

☐ Aadhaar

☐ PAN

☐ Birth Certificate

☐ Passport Photo

Users can manually check completed items.

---

## Feature 5 — Document Information

Clicking any document opens a modal showing:

- Why it is needed
- Where to obtain it
- Issuing authority
- Estimated processing time
- Official link

---

## Feature 6 — Official Links

Every service should include:

- Official application website
- Status tracking page (if available)
- Government portal

Never use unofficial websites.

---

## Feature 7 — AI Summary

At the top of every result page display a short summary.

Example:

"You need four documents before applying for a passport. The process usually takes 2–3 weeks."

---

## Feature 8 — Suggested Questions

Below the search box show example prompts.

Examples:

- I lost my wallet
- I need a passport
- I got married
- My child was born
- I want a driving licence

Clicking one should automatically start the search.

---

## Feature 9 — Responsive Design

The application must work on:

- Mobile
- Tablet
- Desktop

Cards should stack vertically on smaller screens.

---

## Feature 10 — Error Handling

If the AI cannot confidently identify the request:

- Explain the issue.
- Ask a follow-up question.
- Never guess.

Example:

"Do you want to apply for a new passport or renew an existing one?"

---

# MVP Features

The hackathon version must include:

✓ AI Chat

✓ Government Roadmap

✓ Service Cards

✓ Document Checklist

✓ Official Links

✓ Step-by-Step Guidance

Everything else can be added later.
